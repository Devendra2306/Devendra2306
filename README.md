# 🧠 AI Document Intelligence Platform

### *Serverless document processing on AWS, built in phases*

<img src="https://img.shields.io/badge/Status-Phase%201%20Complete-success?style=for-the-badge"/>
<img src="https://img.shields.io/badge/Cloud-AWS-FF9900?style=for-the-badge&logo=amazon-aws"/>

**Overview**
A multi-phase serverless platform for document ingestion and intelligence — drag-and-drop upload UI backed by a fully event-driven AWS pipeline, with document state tracked end-to-end.

### 🏗 Architecture (Phase 1)

```text
        USER
         │
         ▼
  React Upload UI
         │
   S3 Presigned URL
         │
         ▼
    Amazon S3
         │
   Lambda Trigger
         │
         ▼
    DynamoDB
 (UUID + status tracking)
         │
         ▼
   API Gateway
         │
         ▼
   Client Status Poll
```

### 🛠 Tech Stack

| Layer | Technology |
|---|---|
| Frontend | React (drag & drop) |
| Storage | S3 (presigned URLs) |
| Compute | AWS Lambda |
| Database | DynamoDB (UUID doc tracking) |
| API | API Gateway |
| Region | ap-south-1 |

## 🚀 Project Ecosystem

### 🔎 RAG-Based Document Q&A System
Ask natural-language questions over your own documents — retrieval-augmented generation, served through a full-stack app.

**Stack:** `LangChain` → `ChromaDB` → `Gemini API` → `FastAPI` → `React`
**Learned:** vector retrieval pipelines, prompt grounding, chunking strategy, LLM API orchestration

### 🌫️ AQI Monitoring Pipeline
End-to-end ELT pipeline for air quality data — historical batch extraction from S3, live polling from the OpenAQ API, and an interactive dashboard.

**Stack:** `Python` → `DuckDB` → `Dash` → `OpenAQ API`
**Features:** live map view, trend charts, pollutant heatmap, data-freshness monitoring
**Learned:** ELT design, embedded analytics DBs, dashboarding, memory-constrained deployment (Render free tier)

### ☁️ CloudVault — Cloud Storage Platform
A Google Drive-style storage platform with secure file handling and relational data modeling.

**Stack:** `React` → `Node.js` → `PostgreSQL` → `Prisma` → `AWS S3`
**Learned:** auth flows, relational schema design, S3 object storage, ORM-driven backend architecture
